# AI-Customer-Support-System-Salesforce
AI-driven customer support system built on Salesforce Service Cloud using Flow automation and data modeling.

Autonomous AI Customer Support System (Salesforce)

- Overview

This project is an AI-driven customer support system built on Salesforce Service Cloud.
It simulates AI behavior using Flow Automation to classify, prioritize, and resolve customer support cases automatically.


- Problem

Customer support teams often face:

A, High volumes of repetitive cases

B, Slow response times

C, Poor prioritization

D, Limited visibility into case trends


- Solution

This system automates the support process by:

A, Classifying cases based on customer input

B, Assigning priority using sentiment and case type

C, Auto-resolving simple issues

D, Routing complex cases to agents

E, Providing dashboards for insights


- Key Features

A, AI-like case classification using Flows

B, Sentiment-based priority scoring

C, Auto-resolution for simple requests

D, Dynamic case routing

E, AI-generated summaries and responses

F, Real-time dashboards and reporting


- Architecture
- System Architecture

Customer → Case → AI Processing (Flows) → Automation → Resolution → Dashboard


- Data Model

The system uses both standard and custom objects:

- Standard Objects:

1, Account

2, Contact

3, Case

4, Knowledge

- Custom Objects

1, Issue Category

2, Resolution Template

3, Customer Sentiment

4, Customer Health Score

- Automation
- Automation Logic

Flow acts as a decision engine by:

A, Classifying cases using keyword-word based logic

B, Mapping cases to structured categories

C, Calculating priority scores 

D, Triggering auto-resolution or escalation

- Sample Scenarios:

A, Password Reset

1, Auto resolved

2, Low priority

B, Billing Issue

1, Requires agent

2, Medium priority

C, Technical Problem

1, Negative sentiment

2, High priority
